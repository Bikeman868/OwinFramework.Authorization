Owin Framework authorization middleware

The authorization middleware allows other middleware to specify the required permission and have all of the identification and permission checking handled elsewhere. There is nothing more that OWIN Framework middleware has to do beyond specifying the name of the permission that is needed to execute the request.

The authorization middleware in the Owin Framework supports the following configuration options:

Property Default Current Description
documentationRootUrl {documentationRootUrl.default} {documentationRootUrl}

Specifies the URL within the website where documentation will be available for the Authorization middleware.
